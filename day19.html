Training Blog - Day 19

Exploring Redis and brushing up on key backend concepts.

Start of the Day (SOD)

SOD - 15 July 2025

Today’s key goals were:

  • 📦 Learn about Redis and its role in backend development.
  • 📚 Revise previously covered concepts — Node.js, API handling, and deployments.
  • 📝 Write and publish the Day 19 blog.

Task Execution & Challenges

I started the day with a Redis crash course on YouTube. Redis is an in-memory key-value data store, often used for caching, session management, real-time analytics, and pub/sub messaging in modern backend systems.

What stood out was Redis’s speed due to memory-based storage and simple data structures like strings, lists, sets, and hashes. I explored:

  • 🔧 Setting and getting keys using CLI.
  • 🧠 Understanding TTL (time to live) and expiry concepts.
  • 🔄 How Redis integrates with Node.js applications for caching frequently accessed data.

Later, I revised some important concepts like Node.js middleware, route handling, and deployment flow — just to stay sharp and ready for future tasks.

Challenges faced today:

  • 📉 At first, understanding Redis data persistence strategies (RDB vs AOF) was a bit confusing.
  • 🔐 Figuring out how Redis works alongside authentication/session storage required going through real-world examples.

End of the Day (EOD)

  • ✅ Revised key backend topics for reinforcement.

Conclusion

Redis turned out to be a really interesting and powerful tool to learn. It gave me a better understanding of performance optimization and caching. The revision helped solidify my backend fundamentals. Overall, a productive day filled with both learning and reflection.
