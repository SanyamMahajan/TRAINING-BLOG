Training Blog - Day 20

Dived into modern backend tools — GraphQL and Prisma.

Start of the Day (SOD)

SOD - 16 July 2025

Planned tasks for today:

  • 🔍 Learn GraphQL — its syntax, structure, and real-world use cases.
  • 🛠️ Explore Prisma ORM and how it integrates with modern backend stacks.
  • ✍️ Write and publish my Day 20 training blog.

Task Execution & Challenges

I began the day by learning the fundamentals of GraphQL. Unlike REST, GraphQL allows fetching precise data with a single query. I explored how to define types, queries, and mutations, and practiced writing resolvers for custom data fetching.

Next, I explored Prisma — a next-generation ORM for Node.js and TypeScript. It simplifies database management with auto-generated queries and type-safe access. I went through schema definitions, database migrations, and Prisma Studio.

Key concepts I worked with today:

  • 📌 Setting up a basic GraphQL server using Apollo Server.
  • 📋 Writing typeDefs and resolvers to fetch data from a dummy database.
  • 🔁 Implementing Prisma schema and using it to interact with PostgreSQL.
  • 🧪 Testing queries and mutations with GraphQL Playground.

Challenges faced:

  • 🧠 Understanding resolver logic and how it maps to nested queries was tricky at first.
  • ⚠️ Working with Prisma migrations and syncing schema with DB needed attention to detail.
  • 🔄 Switching context between GraphQL and SQL required a clear mental model.

End of the Day (EOD)

  • ✅ Completed GraphQL learning — wrote basic schemas, queries, and mutations.
  • ✅ Explored Prisma and practiced using it for schema and DB operations.
  • ✅ Wrote and published Day 20 blog.

Conclusion

Today's exploration of GraphQL and Prisma added two powerful tools to my backend development arsenal. I now understand how to request structured data flexibly using GraphQL and manage databases efficiently using Prisma. Looking forward to integrating both in future real-world projects!
