Training Blog - Day 1

Starting with GitHub CLI basics and setting up my daily blog structure.

Start of the Day (SOD)

Today began with a message that my tasks would be shared shortly. Meanwhile, I was assigned two key objectives:

  • 📌 Revise and practice all GitHub CLI commands.
  • 📝 Set up and publish a sample daily training blog website.

Both tasks aimed to lay the foundation for the training ahead by reinforcing fundamentals and building daily discipline through documentation.

Task Execution & Challenges

I began revisiting all essential GitHub CLI commands — initializing a repo, cloning, committing, pushing, creating branches, resolving conflicts, and more. I referred to GitHub's official CLI documentation and performed each step on a dummy local repository.

Simultaneously, I designed and launched this training blog website using HTML, CSS, and JavaScript. It’s structured to document my day-wise learning experience throughout the internship.

Challenges faced:

  • Configuring SSH authentication smoothly on CLI for push commands.
  • Understanding how to structure a scalable daily blog UI.
  • Balancing aesthetics with readability for content sections.

These early hurdles helped me get more comfortable with the CLI and frontend structure, and now I feel prepared to take on more technical work.

End of the Day (EOD)

  • ✅ Revised and practiced key HTML & CSS fundamentals.
  • ✅ Completed revision and practical use of GitHub CLI commands.
  • ✅ Designed and published this blog layout.
  • ✅ Understood the training structure and expectations for the coming days.

I’m excited to continue learning, building, and improving each day from here onward.

Time Spent

Tracked using Clockify:

  • Blog Setup: 2 hours 47 minutes
  • HTML & CSS Revision: 1 hour 40 minutes

Conclusion

Day 1 was all about getting hands-on and setting the tone for a productive internship. I not only revised vital version control practices but also created a foundation for documenting my learning consistently. Excited for Day 2!
